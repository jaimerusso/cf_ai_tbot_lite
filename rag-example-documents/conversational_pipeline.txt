Conversational Pipeline:
When a user sends a message via WebSocket, the Worker receives the query and triggers a Conversational Workflow. The workflow starts by fetching the existing dialogue state from the Dialogues Durable Object. It then retrieves the available tools list from the Tool Description Durable Object and sends the full dialogue context along with the tools to Llama 3.3 to determine the user's intent. If the model detects a searchDocument tool call, the query is forwarded to a Search Workflow running concurrently. The Search Workflow generates a vector embedding of the query using BGE Base EN v1.5, searches Vectorize for the top 3 most similar vectors, and returns the matching content as tool results. Back in the Conversational Workflow, the tool results and updated dialogue are sent to Llama 4 again to generate the final response. The dialogue is then updated in the Dialogues Durable Object and the final response is sent back to the user via WebSocket.

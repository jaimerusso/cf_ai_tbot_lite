Document Deletion Pipeline:
When a user requests a file deletion through the UI, the Worker triggers a Delete Workflow with the file name. The workflow fetches the embedding IDs associated with the document from the Documents Durable Object, then deletes those embeddings from Vectorize. It then fetches the current tool description from the Tool Description Durable Object and, using the document's resumee and the current description, calls Llama 3.3 to generate an updated description without the deleted document's information, saving it back to the Tool Description DO. Finally, the document entry is deleted from the Documents Durable Object.

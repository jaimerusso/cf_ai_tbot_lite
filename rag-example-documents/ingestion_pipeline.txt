Document Ingestion Pipeline:
When a user uploads files through the UI, the Worker receives the files and spawns one Ingest Workflow per file, all running concurrently. Each workflow generates a vector embedding of the file content using BGE Base EN v1.5, then inserts the embedding and content into Vectorize. It then sends the content to Llama 3.3 to generate a document resumee. With the resumee and the current tool description fetched from the Tool Description Durable Object, it calls Llama 3.3 again to generate an updated search tool description, which is saved back to the Tool Description DO. Finally, the document metadata is saved to the Documents Durable Object.

Document Ingestion Pipeline:
When a user uploads files through the UI, the Worker receives the files and spawns one Ingest Workflow per file, all running concurrently. Each workflow starts by chunking the file content into smaller pieces. For each chunk, a vector embedding is generated using BGE Base EN v1.5, and the embedding along with the chunk content is inserted into Cloudflare Vectorize, which returns the generated IDs. It then sends the full document content to Llama 3.3 to generate a document resumee. With the resumee and the current tool description fetched from the Tool Description Durable Object, it calls Llama 3.3 again to generate an updated search tool description, which is saved back to the Tool Description DO. Finally, the document metadata along with the embedding IDs is saved to the Documents Durable Object.
